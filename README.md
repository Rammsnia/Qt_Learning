# Qt_Learning
Просто папка с проектами для практики в Qt/C++ и других навыках, которые мне необходимы для работы


# Мне надо знать следующее:
1) Язык С++ (полное обучения от новичка до профи)
2) Практическое освоение программного средства по Qt5
3) Практическое освоение программы версионного хранения исходного кода GitLab
4) Практическое основение систем автоматизации сборки программного обеспечения (cmake, qmake)
5) Redmine для управления проектами
6) ГОСТ 19.001, 19.103, 19.105, 19.301, 19.505


# Годовой план обучения: Разработка ПО для корабельной навигации

## Цели
- Освоить C++, STL, Qt, Bash, GitLab, CMake/QMake, Boost, SQL, PVS-Studio, Redmine
- Изучить ГОСТы по разработке ПО
- Реализовать проекты в области корабельной навигации

## Общая структура
| Этап       | Длительность | Технологии                          | Проекты                          |
|------------|--------------|-------------------------------------|----------------------------------|
| **Базовый**| 1-3 месяца   | C++, STL, Git, Bash                 | Консольные утилиты              |
| **Qt Core**| 4-6 месяцев  | Qt Core, CMake, SQL                 | Обработка навигационных данных  |
| **Продвинутый**| 7-9 месяцев| Qt GUI, Boost, PVS-Studio         | Визуализация карт               |

## Детальный план по месяцам

### Месяц 1: Основы C++ и Git
**Темы:**
- Синтаксис C++: типы данных, функции, ООП
- Git: базовые команды, ветвление
- Bash: файловая система, скрипты

**Практика:**
1. Реализовать класс `GeoPoint` (широта/долгота)
2. Калькулятор расстояний (формула Хаверсина)
3. Скрипт для автоматической сборки проектов

**Проект:**  
Консольная утилита для расчёта дистанции между портами

### Месяц 2: STL и продвинутый C++
**Темы:**
- Контейнеры STL: `vector`, `map`, `set`
- Алгоритмы STL: сортировка, поиск
- Шаблоны и RAII

**Практика:**
1. Парсинг навигационных логов (формат NMEA-0183)
2. Фильтрация данных GPS с использованием STL
3. Кэширование маршрутов

**Проект:**  
Обработчик сырых данных GPS с фильтрацией выбросов

### Месяц 3: Системы сборки и Qt Core
**Темы:**
- CMake/QMake: создание билд-систем
- Qt Core: сигналы/слоты, QObject
- Многопоточность

**Практика:**
1. Сборка проекта с поддержкой Astra Linux
2. Чтение/запись навигационных маршрутов (JSON/QSettings)
3. Потоковая обработка данных AIS

**Проект:**  
Сервис мониторинга судов с использованием AIS-потоков

### Месяц 4: Базы данных и SQL
**Темы:**
- SQL: запросы, JOIN, транзакции
- Qt SQL: интеграция с SQLite
- Оптимизация запросов

**Практика:**
1. Проектирование БД для хранения маршрутов
2. Импорт картографических данных из S-57 стандарта
3. Генерация отчётов по пройденным маршрутам

**Проект:**  
База данных навигационных карт с поиском точек интереса

### Месяц 5: Qt GUI и визуализация
**Темы:**
- Qt Widgets: QGraphicsView, кастомные виджеты
- Qt Quick (QML): анимации
- Рендеринг карт

**Практика:**
1. Отображение маршрутов на холсте
2. Слой для отрисовки навигационных знаков
3. Интерактивная карта с масштабированием

**Проект:**  
Визуализатор морских карт с поддержкой слоёв

### Месяц 6: Boost и безопасность
**Темы:**
- Boost: Asio (сети), Geometry (геометрия)
- PVS-Studio: статический анализ
- Тестирование (Google Test)

**Практика:**
1. Расчёт зон видимости радаров
2. Анализ покрытия кода
3. Оптимизация гео-алгоритмов

**Проект:**  
Модуль расчёта безопасных маршрутов (обход опасных зон)

### Месяц 7: Интеграция и ГОСТы
**Темы:**
- ГОСТ 19.ххх: требования к документации
- Redmine: управление задачами
- CI/CD в GitLab

**Практика:**
1. Оформление ТЗ по ГОСТ 19.201-78
2. Автоматизированная сборка в Docker
3. Регламент тестирования по ГОСТ 19.301-79

**Проект:**  
Система документооборота для навигационного ПО

### Месяц 8: Комплексное приложение (ядро)
**Задачи:**
- Интеграция модулей: карты, БД, расчёты
- Реализация ядра навигационной системы
- Протоколирование по ГОСТ 19.103-77

**Технологии:** C++17, Qt, Boost.Geometry

**Проект:**  
Ядро ECDIS (Electronic Chart Display and Information System)

### Месяц 9: GUI и оптимизация
**Задачи:**

Интерактивная панель управления судном

### Месяц 10: Тестирование и документирование
**Задачи:**
- Нагрузочное тестирование
- Оформление по ГОСТ 19.505-79

### Месяц 11: Внедрение и обратная связь
**Задачи:**
- Пользовательское тестирование

Полевые испытания прототипа

**Задачи:**
- Исправление замечаний

## Рекомендации
1. **Астра Линукс:** Все проекты собирать под SE или Common Edition
2. **Тестирование:** Использовать стенд с реальными GPS-модулями
3. **Документация:** Вести в Redmine с привязкой к ГОСТам
4. **Аппаратное обеспечение:** 
   - Raspberry Pi 4 для тестового стенда
   - Датчики GPS/ГЛОНАСС
   - IMU (Inertial Measurement Unit)

## Идеи для проектов
1. **Модель судового журнала** (C++/Qt/SQL) - Замена бумажным журналам
2. **Планировщик маршрутов** (Boost.Geometry) - Расчёт пути с учётом течений
3. **Трекер целей** (Qt Network) - Анализ AIS-сообщений
4. **Симулятор навигации** (QML/OpenGL) - 3D визуализация судна
5. **Анализатор безопасности** (PVS-Studio) - Проверка кода на соответствие ГОСТ Р 58239-2018
